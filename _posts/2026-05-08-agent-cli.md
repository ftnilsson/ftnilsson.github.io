One thing that I have constantly struggled with is the best way to share agents, skills and prompts. No matter if you are a solo developer, working in a small team or want to apply this on an organisation/ enterprise level I have not found a way that really worked for me. So this is my attempt to fix this.

---

## The Problem

We spend enormous effort crafting the perfect AI coding instructions. We nail the naming conventions, the testing philosophy, the architectural patterns. Then we start a fresh repo and immediately undo all of that work by copy-pasting (or worse, rewriting from memory) the same instructions all over again.

And it gets messier when you factor in that different tools expect instructions in different places:

- GitHub Copilot wants `.github/copilot-instructions.md`
- Cursor wants `.cursorrules`
- Claude wants `CLAUDE.md`

In my experience, this is also how instructions rot. The version in your new project is two months older than your current thinking, and the file sitting in your monorepo is different again. You have no idea which is canonical.

## Enter `agent-cli`

[Git repo - agent-cli](https://github.com/ftnilsson/agent-cli)

`agent-cli` is a CLI tool that pulls AI coding instructions from a central repository into any project — in one command.

```bash
agent init
agent preset nextjs
agent install --target copilot
```

That's it. One source of truth. One command. The right file in the right place.

The `--target` flag handles the format differences between tools:

```bash
agent install --target copilot   # → .github/copilot-instructions.md
agent install --target cursor    # → .cursorrules
agent install --target claude    # → CLAUDE.md
```

No more reformatting. No more "which version is right?"

## Skills Are Composable

One thing I really like about the design: your instructions are modular, not monolithic.

Instead of one enormous blob of rules that covers everything (and therefore clutters the AI's context with irrelevant stuff), you pick exactly the skills you need:

```bash
agent add development/architecture
agent add backend/auth
agent add agents/nextjs
agent install
```

This matters more than it sounds. **Bloated prompts are a real problem** — the more noise you put in, the more signal gets lost. Being able to compose only what's relevant keeps your instructions lean and focused.

## Reproducible Across the Team

This is the part I'd push in front of any tech lead. The manifest file — `.agent.json` — is a lockfile-backed config. Commit it to source control, and every developer on the team runs:

```bash
agent install
```

...and gets exactly the same instructions, pinned to the same version:

```json
{
  "source": "github:ftnilsson/agent-cli",
  "ref": "v1.2.0",
  "include": ["development/architecture", "backend/auth", "agents/nextjs"]
}
```

This is how we treat dependencies. This is how we *should* treat AI instructions.

## See What Changes Before You Commit

`agent diff` is a nice touch. Before any install, you can see exactly what would change:

```
+ agents/nestjs         (new)
~ development/git       (upstream changed)
= backend/auth          (up to date)
```

No surprises. No regressions you discover three days later when the AI starts behaving differently.

## Quick Start

If you want to try it without writing a single config file by hand:

```bash
npx agent-cli init --interactive
```

It walks you through the available skills, lets you pick what fits your stack, and installs everything. Under a minute.

---

## Final Thoughts

Managing AI instructions like copy-paste artefacts is the same mistake we made with shared libraries before package managers existed. We all knew it was bad, and we did it anyway because there was no better option.

Now there is.

If you're on a team and everyone has slightly different AI setups, that's technical debt — and it compounds. The inconsistency in output isn't a quirk of the AI; it's a quirk of your tooling.

So here's my challenge to you: next time you start a new project, don't paste the instructions from the last one. Take 10 minutes to set up `agent-cli`. You'll thank yourself the third time you open a new repo and realise there's nothing to copy.
